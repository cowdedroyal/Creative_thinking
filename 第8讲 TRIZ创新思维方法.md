# 案例 1：研究生论文查重率高的“治标不治本”

某高校发现研究生论文查重率居高不下, 简单采取 “提高查重标准” “重复率超标的论文直接打回”的措施（关注表象：查重率数据），却未分析核心原因：部分导师指导不足、学生缺乏学术写作方法培训、科研选题创新性不足。最终，学生为降低查重率选择“同义词替换”“语序调整”等投机方式，论文质量反而下降，查重率高的问题仍未解决。

# 案例 2：生产线上的“设备故障反复”

某工厂频繁出现设备停机问题，维修团队仅关注“更换故障零件”（表象：零件损坏），却未排查核心原因：设备运行温度过高导致零件加速老化，而温度过高的根源是散热系统设计缺陷。最终，故障零件反复更换，生产成本增加，生产线效率仍未提升。

只关注表象、忽视内在关键要素的核心后果是：决策偏离本质、问题反复发生、资源浪费、错失核心价值，本质是“治标不治本”的逻辑误区。

核心逻辑：透过现象看本质，抓住“因果关系”而非“关联关系”，可通过三步验证：

1. 提问 “为什么”：针对问题 / 现象连续追问 3-5 个 “为什么”，挖掘核心原因（如“查重率高  $\rightarrow$  为什么  $\rightarrow$  学生不会写  $\rightarrow$  为什么  $\rightarrow$  缺乏写作培训  $\rightarrow$  为什么  $\rightarrow$  导师指导不足”）；  
2. 聚焦 “核心价值”：明确目标的核心需求（如“课程创新”的核心是“学生学习效果”，而非形式）；  
3. 验证 “因果闭环”：确保策略直接针对核心要素（如“解决写作能力不足”需配套“写作培训 + 导师一对一指导”，而非“提高查重标准”）。

# 案例 1：追问到底

在日本丰田汽车公司，曾经流行一种管理方法，叫作“追问到底”。也就是说，对公司新近发生的每一件事，都采用追问到底的态度，以便找出最终的原因。一旦找到了最终原因，那就对问题有了深刻的认识。比如，公司的某台机器突然停了，那就沿着这条线索进行一系列的追问。

问: “机器为什么不转了?” 答: “因为保险丝断了。”

问: “为什么保险丝会断?” 答: “因为超负荷而造成电流太大。”

问: “为什么会超负荷?” 答: “因为轴承枯涩不够润滑。”

问: “为什么轴承枯涩不够润滑?” 答: “因为油泵吸不上来润滑油。”

问: “为什么油泵吸不上来润滑油?” 答: “因为抽油泵产生了严重磨。”

问: “为什么油泵会产生严重磨损?” 答: “因为油泵未装过滤器而使铁屑混入。”

追问到此，最终的原因就算找到了。给油泵装上过滤器，再换上保险丝，机器就正常运行了。

# 使用 5Why 分析法的关键要点:

1. 提问原则：从“是什么”到“为什么”，避免假设性结论每一层原因必须基于事实，可验证追问要深入，避免停留在表面（如“操作失误”）  
2.案例启示：制造业问题多源于设备维护、流程设计缺陷服务业问题常涉及管理流程、人员培训技术问题往往隐藏着“矛盾点”，是TRIZ创新突破口  
3. 进阶应用：结合鱼骨图可视化因果链  
用TRIZ理论解决分析中发现的技术矛盾  
建立“问题-分析-解决方案-验证”闭环改进机制

以上案例展示了TRIZ五个WHY分析法如何从表象问题层层深入，最终找到可解决的根本原因。无论哪个行业，掌握这种“打破砂锅问到底”的思维方式，都能有效提升解决复杂问题的能力。

# 第1问：为什么这部分研究生毕业后工作好找？

表面原因：求职时与企业岗位需求的“匹配度高”，且具备企业看重的核心能力（如专业技术、问题解决能力），在面试和筛选中脱颖而出。

# 第2问：为什么他们能实现高岗位匹配度，且具备核心能力？

中间原因：所学专业 / 研究方向属于市场紧缺领域（如人工智能、半导体、生物医药等），且研究生阶段积累了“可落地的实践经验”（而非纯理论知识）。

# 第3问：为什么他们的专业/研究方向紧缺，且能积累实践经验？

中间原因：研究生选题时瞄准了行业实际痛点（而非纯学术探索），且在导师指导下参与了“产学研合作项目”或高质量企业实习，将理论转化为实操能力。

# 第4问：为什么他们能瞄准行业痛点选题，且获得优质实践资源？

深层原因：导师的研究方向长期聚焦产业应用（而非纯基础研究），且导师与行业企业保持稳定合作关系（如联合实验室、横向课题），同时学校为研究生提供了明确的实践支持机制（如实习学分认定、校企合作基地）。

# 第5问：为什么这些导师聚焦产业应用、校企合作紧密，且学校有完善的实践机制？

根本原因：高校的“研究生培养定位”与“资源配置导向”偏向应用型人才培养——学校将“产学研协同”纳入导师考核指标（如横向课题经费、校企合作项目数），同时建立了成熟的“校企合作生态”（如企业导师联合指导、实习基地共建、就业需求反向反馈培养方案），形成“需求-培养-就业”的闭环。

# 小人法

# 01

# 定义

用小人模拟系统中问题或冲突的发生，建立问题模型，激发人们的思维，进而找到问题解决思路的方法；  
聪明的小人（Smart Little People），简称SLP

# 02

# 作用

- 分析问题、建立问题模型时克服技术或专业思维惯性；  
能提供冲突的额外信息，表达出解决问题的一般路径；

# 03

# 应用

当系统内的某些组件不能完成其必要的功能，并表现出相互矛盾的情况时，可以用一组小人来代表这些不能完成特定功能的部件，通过使用能动的小人，实现预期的功能，并根据小人模型对结构进行重新设计。

小人法是 TRIZ 理论中克服思维惯性的核心工具，通过将系统组件想象为能动的“小人”，直观展现系统矛盾并寻找创新解决方案。

# 核心步骤:

1. 分析系统构成，将各组件用不同颜色 / 形状小人表示  
2. 构建问题模型：展示小人如何表现出矛盾行为  
3. 重构小人模型：调整小人位置 / 关系消除矛盾  
4.转化为实际方案：将小人关系映射到技术解决方案

# 矿山爆破装置优化（多点快速连接）

问题：传统爆破装置需手动接通多个接点，效率低且易出错。小人法应用：

1. 系统分析：金属球、接点阵列、电路  
2. 问题建模:

当蓝色小人下滑时，无法与所有黄色小人（接点）准确接触，导致部分电路无法接通  
蓝色小人：代表金属球  
- 黄色小人：代表接点

3. 方案建模:

将蓝色小人（金属球）变为多个由大到小的同心圆环  
使黄色小人（接点）按特定角度排列，确保蓝色小人下滑时依次精准接触

4. 实际方案:

- 将爆破装置接点设计为自上而下逐渐收缩的圆锥面  
- 金属球改为多级同心圆环结构，确保下滑过程中与所有接点顺序接触，实现毫秒级精准爆破

# 航空钛合金管材扩口率提升（材料微观结构优化）

问题：TA18 钛合金无缝管扩口率仅  $30\%$  ，远低于航空工业要求的  $60\%$

# 小人法应用：

1. 系统分析：钛合金管材微观组织结构  
2. 问题建模：受外力（扩口）时，小人之间结合力低，易被分离，导致管材破裂。不同大小、杂乱排列的小人：代表不规则晶粒  
3. 方案建模:

调整小人大小一致，紧密排列成有序结构

使每个小人与相邻小人牢固连接，受力时能均匀分担

4. 实际方案:

优化熔炼工艺：调整浇注系统，获得均匀细小的初始晶粒

改进轧制工艺：采用两辊与多辊组合轧制，控制轧制参数 (Q 值)，形成有利于扩口的织构组织

最终使管材扩口率突破  $60\%$  ，解决了长期依赖进口的技术难题

# STC 算子

# 01 定义

尺度、时间、成本  
单独考虑尺寸、时间、成本的一个因素，而不考虑其他的两个因素。引申的意思就是一个产品有诸多因素组成，单一考虑相应因素，而不是统一考虑

# 02 作用

- 克服物体形状、外观、参数思维惯性  
- 迅速发现对研究对象最初认识的不准确和误差  
$\bullet$  重新认识研究对象

# 03 应用

- 不是为了获取问题的答案，而是为了解放思路  
- 用STC算子思考后，可以发现系统中的技术矛盾或物理矛盾  
克服思维定势，改善思维方式的一种很好的工具

# 全过程包括6种假设初始条件下的思维试验：

1、研究对象的尺寸：无穷大（ $S \approx \alpha$ ）或无穷小（ $S \approx 0$ ）；  
2、研究对象的成本（允许的支出）：无穷多（ $C \approx \alpha$ ）或无穷少（ $C \approx 0$ ）。

# 实例：对船用海锚的STC思维试验

研究对象：船、锚、水和海底在尺寸、时间和成本三个方面的参数进行一系列变化的思维试验。

假设船身长100m，吃水量10m，船距海底1km，锚放到海底需1h的时间。

试验1：尺寸  $S \approx \infty$  把船的尺寸增加为原来的100倍，变为  $10 \mathrm{~km}$ ，这时船底已经触到海底了，船沉到了海底，也就不需要海锚了。

试验2：尺寸  $S \approx 0$  如果船的尺寸缩小为原来的  $1 / 1000$ ，变为  $10 \mathrm{~cm}$ ，问题解决了船变得太小了（如同一块小木片），缆绳的长度和重量远远超过小船的浮力，船将无法控制并将会沉没。

试验3：时间  $T \approx \infty$  当时间为10h的时候，锚下沉得很慢，可以很深地嵌入海底，打下扎到海底的桩子。有一种旋进型的锚，在美国已获得专利的振动锚；电动机的振动将锚深深地嵌入海底（系留力是锚自重的20倍）。但这种方法不适用于岩石海底。

![](https://cdn-mineru.openxlab.org.cn/result/2025-12-13/576e2eb7-9bf4-47fc-98c1-e20377374276/0118e4bbf4944ac96957140782b4af2a2d8cc7d3021df8feefb8503ce3ede446.jpg)

试验4：时间  $T \approx 0$  如果把时间缩减为原来的1/100，就需要非常重的锚，使它能够快速地沉降到海底。如果时间缩减为原来的1/1000，锚就要像火箭一般降入海底。如果缩减为原来的1/10000，那么只能利用爆破焊接，将船连接到海底了。

试验5：成本  $C \approx \infty$  如果允许不计成本，就可以使用特殊的方法和昂贵的设备（用白金做锚，利用火箭、潜水艇和深海潜箱）

试验6：成本  $C \approx 0$  如果需要的成本为零，那就利用不需要花费成本，现有的环境资源——海水。如果可行的话，则可以被认为是最好的方法。

# 九屏幕法的核心功能（适配创新全流程需求）

# 1. 打破思维惯性，拓展分析边界

- 功能本质：突破“仅关注当前系统本身”的固化思维，从时间和空间两个维度延伸分析视野，避免“头痛医头、脚痛医脚”的局部优化。  
- 应用场景：当科研或工程中遇到“传统方法无法突破的瓶颈”时（如“如何提升无人机续航”“如何优化实验室反应效率”），九屏幕法可引导从超系统（如无人机的能源供给超系统）、过去技术（早期燃油无人机）、子系统（电池组件）等维度寻找新思路。

# 2. 全面识别系统矛盾与隐藏资源

- 功能本质：通过九域全覆盖，暴露系统与子系统、超系统之间的隐性矛盾（如“手机续航”与“电池重量”的矛盾，需关联超系统的充电设施、子系统的芯片功耗），同时挖掘未被利用的资源（如“利用人体体温为智能穿戴设备供电”，即子系统与超系统的能量资源整合）。

应用场景：产品创新中“功能升级与成本控制”的矛盾（如高校实验室设备改造，需在不增加预算的前提下提升精度），可通过九屏幕法从“过去的低成本原型”“超系统的共享资源”中寻找解决方案。

# 3. 预测技术发展趋势，指导长期创新

- 功能本质：通过分析“系统过去  $\rightarrow$  现在  $\rightarrow$  未来”的演化轨迹，结合超系统、子系统的发展规律，预测技术的下一步迭代方向，避免短期创新局限。  
- 应用场景：科研课题选题（如“新能源材料研发”），可通过九屏幕法分析材料的历史技术（传统化石能源材料）、当前状态（现有新能源材料）、未来趋势（高效环保材料），以及超系统（能源供给系统）、子系统（材料微观结构）的需求，确定具有前瞻性的研究方向。

# 4. 系统化梳理问题，提升解决方案的完整性

- 功能本质：九屏幕法提供标准化的分析框架，避免遗漏关键影响因素，确保解决方案覆盖“系统－子系统－超系统”“过去－现在－未来”的全维度，提升方案的可行性与稳定性。  
- 应用场景：工程系统优化（如“智能温室控制系统改造”），需考虑超系统（气象环境）、系统本身（温控设备）、子系统（传感器组件）的过去（传统温室）、现在（半自动温室）、未来（全智能温室），确保改造方案适配不同环境与技术阶段。

# 金鱼法案例解决步骤

- 四根火柴棒只是四条线段，而组成一个“田”字至少需要六条线段，并且火柴棍不能折断。  
在什么情况下，幻想部分可变为现实？  
- 借助它物；火柴棍上自身含有组成“田”字的资源。

# 04 确定系统、超系统和子系统的可用资源。

- 超系统：火柴盒、桌面、空气、重力、灯光等；  
- 现系统：四根火柴棍；  
- 子系统：火柴棍的横端面和纵端面。

- 四根火柴棍借助火柴盒或者桌角的两条边就能摆成一个“田”字；  
- 四根火柴棍借助两条直光线也可以组成一个“田”字；  
- 火柴棍的横断面是个矩形，而四个矩形就能组成一个“田”字。

# 核心差异与适用场景快速决策

# 1. 按“创新阶段”选择

- 问题诊断阶段：优先用「5why法」（找根本原因）+「九屏幕法」（拓展问题边界，避免遗漏关联因素）；  
- 目标设定阶段：优先用「最高理想度法」（设定终极目标）+「九屏幕法」（结合趋势预测，确保目标前瞻性）；  
- 概念生成阶段：优先用「金鱼法」（突破性概念）+「九屏幕法」（挖掘资源，支撑概念落地）；  
- 方案落地阶段：优先用「小人法」（具象化结构／功能优化）+「矛盾矩阵」（解决落地中的具体矛盾）。

# 2. 按“问题类型”选择

- 根源不明的问题：5why 法（如“实验结果异常”“设备故障”）；  
- 系统优化类问题：最高理想度法 + 九屏幕法（如“提升产品性能”“降低成本”）；  
- 结构 I 组件冲突问题：小人法（如“部件干涉”“功能冲突”）；  
- 突破性创新问题：金鱼法（如“从零到一的新产品”“行业技术革新”）；

科研选题 / 趋势预测：九屏幕法（如“跨领域课题定位”“技术演进方向判断”）。

# 协同应用案例（智能农业灌溉系统创新）

# 1. 问题诊断

- 用「5why法」溯源：灌溉效果差  $\rightarrow$  水量分配不均  $\rightarrow$  传感器监测不准  $\rightarrow$  安装位置不合理  $\rightarrow$  未考虑作物生长周期差异（根本原因）；  
- 用「九屏幕法」拓展：梳理“超系统（农田生态、气象系统）－系统（灌溉设备）－子系统（传感器、水泵）”的过去（传统人工灌溉）、现在（半自动灌溉）、未来（智能自适应灌溉），挖掘资源（如气象数据、作物生长数据）。

# 2. 目标设定

- 用「最高理想度法」设定 IFR：“无需人工干预，根据作物需求、气象条件自动精准灌溉，零水资源浪费、零能耗、低成本”。

# 3. 概念生成

·用「金鱼法」提出幻想方案：“作物自己‘告诉’灌溉系统需要多少水”；  
- 拆分幻想：可行（传感器监测作物含水量）+ 不可行（作物直接“沟通”） $\rightarrow$ 转化为“通过传感器监测作物叶片湿度 + 土壤含水量，结合气象数据自动决策”。

# 4. 方案落地

- 用「小人法」优化结构：将“传感器、水泵、管道”拟人化为“监测小人、供水小人、输送小人”，通过调整“小人协作逻辑”（如传感器实时反馈  $\rightarrow$  水泵精准启停  $\rightarrow$  管道分段供水），解决水量分配不均问题。

# 5. 最终方案

智能自适应灌溉系统：集成作物叶片湿度传感器 + 土壤传感器 + 气象数据接口，通过 AI 算法自动调整灌溉量与时间，水资源利用率提升  $40\%$  ，能耗降低  $30\%$  ，完全适配高校农业工程专业的科研课题与教学实践。
