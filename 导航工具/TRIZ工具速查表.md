## 一、问题分析工具

### 🔍 5个WHY（根因分析法）

**📋 工具卡片**

```
┌──────────────────────────────────┐
│ 🛠️ 工具：5个WHY                   │
├──────────────────────────────────┤
│ 📍 所在章节：第8讲                 │
│                                   │
│ 🎯 适用场景：                      │
│   • 故障诊断                       │
│   • 根本原因分析                   │
│   • 问题追溯                       │
│                                   │
│ ⚡ 核心方法：                      │
│   连续追问5次"为什么"              │
│   直到找到根本原因                 │
│                                   │
│ 📝 操作步骤：                      │
│   1. 描述问题现象                  │
│   2. 问"为什么会这样？"            │
│   3. 针对答案继续问"为什么？"      │
│   4. 重复至少5次                   │
│   5. 找到根本原因                  │
│                                   │
│ 💡 典型案例：                      │
│   • 机器停机分析                   │
│   • 丰田生产方式                   │
│   • 论文查重问题                   │
│                                   │
│ ⚠️ 注意事项：                      │
│   • 不要停在表面原因                │
│   • 每次追问要针对上一个答案        │
│   • 5次是参考，可能更多或更少       │
│                                   │
│ 🔗 相关工具：                      │
│   因果分析、鱼骨图                 │
└──────────────────────────────────┘
```

**📊 应用模板**

| 层次 | 问题 | 答案 |
|-----|------|------|
| 现象 | 发生了什么？ | （描述问题表现） |
| WHY 1 | 为什么会这样？ | （第一层原因） |
| WHY 2 | 为什么会出现上述原因？ | （第二层原因） |
| WHY 3 | 为什么会出现上述原因？ | （第三层原因） |
| WHY 4 | 为什么会出现上述原因？ | （第四层原因） |
| WHY 5 | 为什么会出现上述原因？ | （根本原因） |
| 对策 | 如何解决根本原因？ | （解决方案） |

### 📱 九屏幕法（9-Windows）

**📋 工具卡片**

```
┌──────────────────────────────────┐
│ 🛠️ 工具：九屏幕法                 │
├──────────────────────────────────┤
│ 📍 所在章节：第8讲                 │
│                                   │
│ 🎯 适用场景：                      │
│   • 系统全面分析                   │
│   • 产品进化预测                   │
│   • 战略规划                       │
│                                   │
│ ⚡ 核心方法：                      │
│   时间维度（过去/现在/未来）       │
│   × 系统层级（子/系统/超系统）     │
│   = 9个视角全面分析                │
│                                   │
│ 📝 九屏幕矩阵：                    │
│                                   │
│   超系统│ 过去 │ 现在 │ 未来       │
│   系统  │ 过去 │ 现在 │ 未来       │
│   子系统│ 过去 │ 现在 │ 未来       │
│                                   │
│ 💡 典型案例：                      │
│   • 产品生命周期分析               │
│   • 技术发展趋势预测               │
│   • 科研方向规划                   │
│                                   │
│ ⚠️ 注意事项：                      │
│   • 填写要具体不要空泛              │
│   • 关注横向和纵向的联系            │
│   • 从现在推测未来                 │
│                                   │
│ 🔗 相关工具：                      │
│   S曲线、技术系统进化法则          │
└──────────────────────────────────┘
```

**📊 应用模板**

|          | 过去 | 现在 | 未来 |
|----------|------|------|------|
| **超系统** | 所在大环境的过去 | 所在大环境的现在 | 所在大环境的未来 |
| **系统**   | 系统的过去形态 | 系统的现状 | 系统的未来发展 |
| **子系统** | 关键部件的过去 | 关键部件的现在 | 关键部件的未来 |

**分析要点：**
- 横向看：系统如何随时间进化
- 纵向看：不同层级如何相互影响
- 对角看：超系统的未来会如何影响子系统

## 二、矛盾解决工具

### ⚡ 物理矛盾 - 4种分离原理

#### 原理对比表

| 分离原理 | 核心思想 | 应用场景 | 典型案例 | 关键词 |
|---------|---------|---------|---------|--------|
| ⏰ **时间分离** | 不同时间满足不同需求 | 需求随时间变化 | 折叠自行车 | 时序、阶段、周期 |
| 📍 **空间分离** | 不同空间满足不同需求 | 需求随位置变化 | 飞机垂尾、立交桥 | 位置、区域、层次 |
| 🎚️ **条件分离** | 不同条件满足不同需求 | 需求随环境变化 | 跳水池打气泡 | 温度、压力、速度 |
| 🔄 **系统级别分离** | 整体与部分需求不同 | 系统层级矛盾 | 空中加油机 | 整体、局部、层级 |

## 技术系统八大进化法则（完整版）

| 编号 | 法则名称 | 核心内容 | 应用指导 | 典型案例 |
|-----|---------|---------|---------|---------|
| 1 | **完备性法则** | 系统必须包含：动力、传动、执行、控制4要素 | 检查系统是否完整 | 汽车系统 |
| 2 | **能量传递法则** | 能量必须在系统中顺畅传递 | 优化能量流动 | 减少摩擦损耗 |
| 3 | **协调性法则** | 各子系统要协调配合 | 提高匹配度 | 齿轮啮合 |
| 4 | **提高理想度法则** | 不断提升理想度 | 增功能、减成本 | 产品迭代 |
| 5 | **动态性法则** | 从刚性向柔性发展 | 增加可调节性 | 可调座椅 |
| 6 | **子系统不均衡进化** | 各部分发展不均衡 | 找瓶颈突破 | 木桶短板 |
| 7 | **向微观级进化** | 从宏观向微观发展 | 纳米技术 | 芯片制程 |
| 8 | **向超系统跃迁** | 单系统→多系统组合 | 系统集成 | 智能家居 |

```
遇到问题
    ↓
┌─────────────────────┐
│ 问题类型是什么？     │
└─────────────────────┘
    ↓
    ├─→ 不知道根本原因？
    │       ↓
    │   使用【5个WHY】
    │
    ├─→ 系统复杂难以理解？
    │       ↓
    │   使用【小人法】或【九屏幕法】
    │
    ├─→ 有物理矛盾（同一参数对立需求）？
    │       ↓
    │   使用【4种分离原理】
    │   ├─ 时间相关 → 时间分离
    │   ├─ 位置相关 → 空间分离
    │   ├─ 条件相关 → 条件分离
    │   └─ 层级相关 → 系统级别分离
    │
    ├─→ 有技术矛盾（改善A恶化B）？
    │       ↓
    │   使用【矛盾矩阵】+【40条发明原理】
    │
    ├─→ 需要预测发展趋势？
    │       ↓

    │
    ├─→ 需要寻找最优方案？

            ↓
        使用【金鱼法】
```

## 五、综合应用案例

### 案例1：折叠自行车设计

**问题描述：** 自行车既要坚固稳定（骑行时），又要轻便小巧（携带时）

**问题分析：**
- 类型：物理矛盾（刚性 vs 柔性）
- 参数：结构强度

**工具选择：** 物理矛盾 → 时间分离原理

**解决方案：**
- 使用时：展开，结构稳固
- 携带时：折叠，体积小巧
- 通过铰链机构实现快速转换

**效果：** 完美解决矛盾，创造新市场

---

### 案例2：飞机垂尾优化

**问题描述：** 垂尾既要垂直（稳定性好），又要倾斜（阻力小）

**问题分析：**
- 类型：物理矛盾（角度）
- 参数：垂尾倾斜角

**工具选择：** 物理矛盾 → 空间分离原理

**解决方案：**
- 上部：保持垂直（提供稳定性）
- 下部：适当倾斜（减小阻力）
- 采用渐变过渡设计

**效果：** 稳定性与低阻力兼得

---

### 案例3：实验设备故障分析

**问题描述：** 实验设备频繁停机，影响进度

**问题分析：**
- 需要找到根本原因

**工具选择：** 5个WHY

**分析过程：**
1. WHY1：为什么停机？→ 温度过高保护
2. WHY2：为什么温度高？→ 散热不良
3. WHY3：为什么散热不良？→ 散热风扇不转
4. WHY4：为什么风扇不转？→ 风扇卡住
5. WHY5：为什么卡住？→ 灰尘积累 **←根本原因**

**解决方案：** 定期清理+增加防尘罩

## 七、学习建议

1. 先学会使用【5个WHY】，这是最简单实用的工具
2. 尝试用【小人法】分析一个简单系统

## 八、常见误区

### ❌ 误区1：工具万能论
**错误想法：** TRIZ能解决所有问题
**正确认识：** TRIZ是思维工具，需要结合实际情况

### ❌ 误区2：生搬硬套
**错误做法：** 机械套用矛盾矩阵
**正确做法：** 理解原理，灵活应用

### ❌ 误区3：忽视简单方法
**错误想法：** 一定要用复杂的TRIZ工具
**正确认识：** 简单问题用简单方法（如5个WHY）

### ❌ 误区4：只学理论不实践
**错误做法：** 只看不练
**正确做法：** 每学一个工具就找实际问题练习
