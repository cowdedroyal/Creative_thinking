# 理想化模型：

所涉及的要素包括理想系统、理想过程、理想资源，理想方法，理想机器，理想物质等等。

理想系统：没有实体，没有物质，也不消耗资源，但能实现所需要的功能。

理想过程：只有过程的结果，而没有过程本身，突然获得所需要的结果。

理想资源：存在无穷无尽的资源，供随意使用，而且不必受其他条件约束。

理想方法：不消耗能量和时间，只是通过自身调节，获得所需的功能。

理想机器：没有质量、体积，但能完成所需要的工作。

理想物质：没有物质，但功能却得以实现。

孙子说：不战而屈人之兵！

技术系统是人类为了实现某种功能而设计、制造出来的一种人造系统，在技术系统使用和改进的过程中，其优劣需要进行评价和比较。

例如，买笔记本电脑，评估性价比。

技术系统能够提供一个或多个有用功能，也会附带若干我们不希望出现的副作用，称为有害功能。同时，实现技术系统必须要付出一定的时间、空间、材料、能量等成本。（好东西什么都好，唯一的缺点就是贵！）

因此，评价技术系统的优劣可以用系统实现的有用功能 / （有害功能 + 成本）的比值进行衡量，称为技术系统的理想度。

$$
\text {理 想 度} = \frac {\sum \text {有 用 功 能}}{\sum \text {有 害 作 用} + \text {成 本}}
$$

提高系统的理想度:

1. 增加有用功能的数量，或者提升现有有用功能的质量。  
2. 减少有害功能的数量，或者减低现有有害功能的危害。  
3. 减小系统的体积和重量，降低系统实现功能所需的时间、能量，以及充分利用系统内可用资源，利用自然界已有的资源、现象以及科学效应。

极简主义！利用现有资源，使结构越简单、成本越低、效率越高。

随着技术系统的不断进化，其理想度会不断提高，极限的情况是系统的有用功能趋向于无穷大（实际上会增加成本），有害功能和成本则趋近于零（实际上不可能达到），二者的比值（即理想度）为无穷大（权衡利弊博弈）。

# 寻求最终理想解的系统化思路

1. 精确地描述系统中现存的问题和矛盾  
首先要明确系统到底要解决什么问题，即确定目标。  
2. 明确系统所要实现的最根本功能（SVOP：系统 + 动作 + 对象 + 参数）

无论系统如何改进，有用的功能，尤其是最基本的有用功能，是只能加强，不能被改变或者弱化的。  
3. 在明确问题和功能的基础上，思考实现（以上）功能的理想情况

首先考虑有用功能，有害功能和成本全部都为零。也就是说完全不需要整个功能了。  
3-1 需要 / 存在这种功能的终极目的到底是什么？是否可以通过其他方式达成同样目的而使得这种功能不再被需要

接下来考虑有害功能和成本都为零，即不需要现有系统了，但有用功能还是会实现的。

3-2 是否可以不需要系统

a) 对象是否可以自服务，即对象自己实现所需功能  
b) 所需功能是否可由超系统实现  
c) 所需功能是否可由更廉价的其他系统实现

把有害功能降为零

3-3 是否可去除有害作用

a) 是否可剪裁产生有害功能的组件或子系统  
b) 是否可将有害功能配置到超系统中去

把成本降为零

3-4 是否可降低成本

是否可利用系统内部的剩余资源或引入系统外部的“免费”资源，帮助实现消除有害功能或实现有用功能

4. 看其他行业是否已解决本问题  
5. 构建解决方案

任何一种产品、工艺或技术都在随着时间，向着更高级的方向发展和进化，并且它们的进化过程都会经历相同的几个阶段。

阿奇舒勒及其他TRIZ研究者通过分析总结大量的专利信息发现：

每个产品、工艺或技术，都在随着时间向着更高级的方向发展和进化，其规律满足一条S形的曲线。

一般都要经历四个阶段：

婴儿期、成长期、成熟期、衰退期

如：在飞机这一技术系统中，飞机的速度、安全性、载重量、航程等都是其重要的性能参数。

# 1. 婴儿期：（用华为5G举例）

为了回应人们对某种功能的需求，新的技术系统得以开发，这个过程也往往伴随着少部分高级别发明的出现。

此时的技术系统本身结构还不尽成熟，为其提供支持的子系统和超系统也没有完善，因此经常表现出效率低、可靠性差等一系列问题，这在性能指标上有所体现。

同时，为了解决新系统中存在的主要技术问题，需要消耗大量的人力，物力，财力等资源，经济效益普遍为负值。

# 2. 成长期：

在克服婴儿期的起步阻力之后，技术系统进入迅速发展的成长期。

从婴儿起向成长期过渡的标志是一个相对高级别发明的引入，对系统的改进做出了明显的贡献，

从而使系统的性能得以迅速提升,

伴随着经济效益的大幅增加。

对系统的改进转变为小修小补，发明级别逐步下降，发明数量则是稳中有升。

# 3. 成熟期：

成长期大量技术和资源的投入，使得系统日趋完善，步入成熟期，

其性能基本达到了最高水平，可能已经建立了相应的技术标准体系，

伴随而来的是可观的经济效益。

然而系统的发展潜力已经得到充分开发，本阶段则依靠大量低级别的发明，

对系统进行优化和改进，但是对性能的提升作用不明显。（应该提早进行新技术新产品开发，研发是永不止息的，反例：诺基亚，误判进化趋势，错误研发方向）

# 4. 衰退期：

盛极而衰是自然界的基本规律，对于技术系统也是如此。

从成熟期逐步迈入衰退期，系统所采用的技术已经发展到极限，对其进行的改进也基本停滞，表现为专利数量和级别的迅速下降，

系统性能也逐步下滑。

与此同时，该系统所提供的功能相对陈旧，面临着市场的淘汰，或被新开发的技术系统所取代，因而经济效益产生滑坡。

# 法则 1：完备性法则

要实现某项功能，一个完整的技术系统必须包含以下四个部件：动力装置、传输装置、执行装置、控制装置。

完整的技术系统

A. 系统如果缺少其中的任一部件, 就不能成为一个完整的技术系统  
B. 如果系统中的任一部件失效, 整个技术系统也无法 “幸存”

完备性法则有助于确定实现所需技术功能的方法，并节约资源，利用它可对效率低下的技术系统进行简化。

# 应用案例：智能农业灌溉系统升级

- 原始问题：传统灌溉系统仅含水泵（动力）和水管（传输），需人工判断灌溉时机与水量，存在“缺水减产”或“过灌烂根”问题。  
- 进化路径：

a. 补充执行装置：新增旋转式喷头（精准作用于作物根系）；  
b.增设控制装置：嵌入土壤湿度传感器与智能控制器；  
c.优化协同逻辑：传感器（检测湿度）  $\rightarrow$  控制器（对比阈值）  $\rightarrow$  水泵（自动启停）  $\rightarrow$  喷头（调节流量）。

- 实施成效：灌溉效率提升  $60\%$  ，水资源消耗减少  $45\%$  ，作物产量稳定提升  $12\%$  ，完全替代人工操作。

# 教学场景延伸：虚拟仿真实验系统

基础版仅含实验软件（执行）与电脑（动力），升级时补充：

- 数据采集模块（传输）：实时捕捉操作数据；  
- 智能评分系统（控制）：自动判断操作正误并反馈，形成“动力 - 传输 - 执行 - 控制”完整闭环，实验教学覆盖率从  $40\%$  提升至  $100\%$  。

# 法则 2：能量传递法则

(->现代TRIZ转变为流进化法则包含能量流、物质流、信息流->物流、网络、通信等)

A. 技术系统要实现其功能, 必须保证: 能量（流）能够从能量源, 流向技术系统的所有元件。

如果技术系统中的某个元件不接收能量（流），它就不能发挥作用，那么整个技术系统就不能执行其有用功能，或者有用功能的作用不足。

# 法则 6：子系统不均衡进化法则

# 每个技术系统，都由多个实现不同功能的子系统组成。

子系统不均衡进化法则是指:

A. 任何技术系统所包含的各个子系统, 都不是同步、均衡进化的, 每个子系统都是沿着自己的 S 曲线向前发展。  
B. 这种不均衡的进化, 经常会导致子系统之间的矛盾出现。  
C. 整个技术系统的进化速度, 取决于系统中发展最慢的子系统的进化速度。(短板效应)

所以，掌握了该法则，可以帮助人们及时发现并改进最不理想的子系统。

# 法则 6：子系统不均衡进化法则

# 实例：自行车的进化

早在十九世纪中期，自行车还没有链条传动系统，脚蹬直接安装在前轮轴上。

此时，自行车的速度与前轮直径成正比。

因此为了提高速度，人们采用增加前轮直径的方法。但是一味的增加前轮直径，会使前后轮尺寸相差太大，从而导致自行车在前进中的稳定性很差，很容易摔倒。

后来，人们开始研究自行车的传动系统，在自行车上装上了链条和链轮，用后轮的转动来推动车子的前进，且前后轮大小相同，以保持自行车的平衡和稳定。

# 技术系统进化法则的应用

产生市场需求：进化法则具有普适性，可以帮助市场调查人员和设计人员，从进化趋势确定产品的进化路径，引导用户提出基于未来的需求，转变为QFD(质量功能展开/质量屋)的输入，实现市场需求的创新。

定性技术预测：针对目前的产品，可以为研发部门提出如下的预测：

1、对于婴儿期及成长期的产品，在结构参数上进行优化，促使其尽快成熟，为企业带来利润同时也应该尽快申请专利，进行产权保护，使企业在今后的市场竞争中处于有利的位置。  
2. 对于处于成熟期或衰退期的产品，避免进行改进设计的投入或进入该产品领域，同时应关注于开发新的核心技术，以替代已有的技术，推出新一代的产品，保持企业的持续发展。  
3、明确符合进化趋势的技术发展方向，避免错误的投入。

产生新技术：可以对当前产品进行分析，找出更合理的功能实现结构，帮助设计人员完成对系统或子系统基于进化的设计。

专利布局：对于当前还没有市场需求的技术，可以事先进行有效的专利布局，以保证企业未来的长久发展空间和专利发放所带来的可观收益。拥有专利权也可以与其他公司进行专利许可使用的互换，以节省资源，节省研发成本。

选择企业战略制定的时机：一个企业也是一个技术系统，一个成功的企业战略能够将企业带入一个快速发展的时期，完成一次S曲线的完整发展过程。企业需要在成熟期开始着手进行下一个战略的制定和实施，从而顺利完成下一个S曲线的启动，将企业带向下一个辉煌。

# 八大法则应用的核心逻辑总结

1. 预测方向：通过法则预判系统进化路径（如向微观、超系统进化），提前布局创新；  
2.定位问题：用不均衡、协调性法则找到系统短板与错配点；  
3. 设计方案：结合动态性、理想度等法则优化解决方案；  
4. 验证成效：以能量传递、完备性法则检验系统完整性与效率。企业与教育机构应用表明，遵循八大法则可使创新成功率提升3-5倍，研发周期缩短  $40\% - 60\%$  ，是技术创新的“导航地图”。

B. 提高理想度进化路线是技术系统进化的终极方向  
C. 动态性提升进化路线仅关注系统结构的柔性, 不涉及可控性  
D. 子系统协同进化路线强调 “补短板” 而非 “强长

1. 答案：B 解析：“单－双－多系统”进化路线的核心是通过“复制相同组件（微腔）→构建双系统（双微腔）”提升性能，题干中双微腔替代单微腔的设计完全契合该路线；A 侧重功能价值比优化，C 关注系统柔性与可控性，D 聚焦子系统短板升级，均与题干不符。  
2. 答案：C 解析：提高理想度进化路线通过“增加有用功能（数字印刷的个性化定制）、削减有害功能（解决泥活字易损问题）、降低成本（数字印刷成本大幅下降）”优化“有用功能/(成本 + 有害功能)”比值，题干中活字印刷的迭代完全遵循这一逻辑；A关注系统适应性，B侧重融入超系统，D聚焦子系统协同，均不符合。  
3. 答案：B 解析：动态性提升进化路线的核心是“系统从刚性固定向柔性可调、可控性提升进化”，王祯转盘活字通过“动态取字－排版”联动提升可控性与调节效率，符合该路线；A体现“单－双－多系统”路线，C体现子系统协同路线，D体现提高理想度路线。

4. 答案：C 解析：子系统协同进化路线的核心是识别并优先升级“进化最慢的短板子系统”，实现各子系统进化速度匹配，突破整体性能瓶颈；A 是“单－双－多系统”路线的核心，B 是动态性提升路线的核心，D 是提高理想度路线的核心。  
5. 答案：A 解析：该方案通过“增加一层发光单元（复制原有组件）”从“单系统（单层发光层）”向“双系统（双层发光层）”进化，分摊功能以解决矛盾，符合“单-双-多系统”进化路线的逻辑；B关注系统柔性，C侧重元件微型化，D聚焦价值比优化，均不匹配。  
6. 答案：C 解析：动态性提升进化路线既关注“系统结构柔性”，也强调“可控性提升”（如现代数字排版的即时参数调节、自适应适配），“仅关注柔性不涉及可控性”的表述错误；A、B、D 均是对 TRIZ 进化路线的正确描述。
